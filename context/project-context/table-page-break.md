# Перенос таблицы на следующую страницу

## Поведение

- Таблица может разбиваться на части с подписью «Продолжение таблицы» и разрывом страницы.
- Разрыв делается **до** строки, которая не помещается на текущей странице.

## Условие переноса (md2gost/renderable/table.py)

- Перенос выполняется, только если **сумма** высоты уже добавленных строк и высоты текущей строки превышает оставшееся место на странице:  
  `(table_height + row_height) > layout_state.remaining_page_height`.
- Раньше проверялось только `row_height > remaining_page_height`, из‑за чего строка, ещё помещающаяся после уже выведенных, переносилась на новую страницу.
- Дополнительно: перенос не делается, если на текущей странице ещё нет ни одной строки (`table_height > Pt(0.5)`), чтобы не получать пустую таблицу на странице.

## Тесты

- `tests/test_table_break.py` — проверка, что при нехватке места появляется ровно одна подпись «Продолжение таблицы» и две части таблицы (по одной строке на странице). Тест помечен `@unittest.skipUnless(_has_matplotlib, ...)`, так как рендер заголовка/таблицы использует расчёт высот через matplotlib.
