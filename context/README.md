# Context — работа в формате Ralph-цикла

Эта директория содержит артефакты для совместной работы с ассистентом в формате **Ralph-цикла** (plan → implement → verify → document).

## Структура

| Файл/директория | Назначение |
|-----------------|------------|
| **master-promt** | Промпт-инструкция для ассистента: как работать с задачами, тестами и контекстом |
| **tasks-todo/** | Очередь задач. Задачи — файлы `task-NNN.md` с описанием и критериями приёмки |
| **tasks-done/** | Выполненные задачи: после закрытия задача **переносится** сюда из tasks-todo |
| **project-context/** | Директория с файлами контекста по проекту. Название каждого файла явно отражает содержимое (архитектура, списки, тесты и т.д.); ассистент обращается к ним за контекстом |

## Workflow

1. **Постановка задачи** — пользователь или ассистент создаёт/дополняет файл в `tasks-todo/` (например `task-001.md`).
2. **Реализация** — ассистент выполняет задачу **end-to-end**: код → автотесты → отладка (прогон тестов).
3. **После выполнения** — задача переносится в `tasks-done/`, контекст фиксируется в `project-context/` (новые/обновлённые файлы с говорящими именами).

Используй **master-promt** при каждой сессии для согласованного поведения.
